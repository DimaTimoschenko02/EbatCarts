# SmashKarts Clone

Браузерная 3D мультиплеер-игра для компании друзей. Клон SmashKarts.io с расширенной статистикой.

## Стек
- **Движок:** Godot 4.3+, GDScript
- **Мультиплеер:** WebSocketMultiplayerPeer (Godot High-Level Multiplayer API)
- **Сервер:** Godot headless — запускается локально у хоста
- **Клиент:** HTML5 export → браузер
- **Туннель:** ngrok (порт 4444 = WebSocket игры)

## Запуск для разработки
1. Открыть проект в Godot 4
2. Запустить сцену `scenes/lobby.tscn`
3. Одно окно: Host Game → вводит имя
4. Второе окно (или другой браузер): Join с IP `127.0.0.1`

## Ключевые файлы

| Файл | Роль |
|------|------|
| `scripts/network_manager.gd` | Autoload — WebSocket сервер/клиент |
| `scripts/game_manager.gd` | Autoload — HP, kills, deaths, respawn |
| `scripts/player_data.gd` | Autoload — имя локального игрока |
| `scripts/kart_controller.gd` | VehicleBody3D — физика, стрельба, синхронизация |
| `scripts/rocket.gd` | Area3D — движение ракеты, взрыв, урон |
| `scripts/weapon_pickup.gd` | Area3D — подбор оружия |
| `scripts/arena_builder.gd` | Процедурная генерация арены (CSG-примитивы) |
| `scripts/game_world.gd` | Главный скрипт игровой сцены, спавн картов |
| `scripts/hud.gd` | Очки, HP-бар, kill feed |

## Архитектура мультиплеера
- Сервер **авторитарный** для урона и смертей (`GameManager` запускается на сервере)
- Позиции картов синхронизируются через RPC (клиент → все) на частоте 20 Hz
- Ракеты спавнятся по команде сервера на всех клиентах одновременно
- Карты спавнятся через `MultiplayerSpawner` — новые игроки автоматически получают актуальное состояние

## Управление
- **W/S** — газ/тормоз
- **A/D** — поворот
- **Space или ЛКМ** — выстрел (если есть оружие)

## Принципы разработки

- **Визуал и физика — приоритет.** Поведение машинки, ощущение от управления, анимации взрывов и эффекты должны быть близки к оригинальному SmashKarts.io. Всегда думать над лучшим решением, а не быстрым.
- **Arcade feel, не симулятор.** Дрифт, ускорение, повороты — всё должно быть отзывчивым и приятным, как в аркадных карт-играх.
- **Планируется система типов машинок:** большие (медленные, высокий урон/HP), маленькие (быстрые, ловкие, меньше HP). Физические параметры карта делать параметризованными через `@export`, не хардкодить константы.
- **Дрифт** планируется переписать на RigidBody3D с ручной arcade-физикой (не VehicleBody3D) — для точного контроля поведения при заносе.
